## 1. 背景

前三章的主线已经成形：

- `00` 负责前置坡道
- `01` 负责 Warp 执行模型
- `02` 负责进入 Newton 本体

当前 `00` 的问题不是不清楚，而是“刚体与机器人动力学最小词汇”仍偏压缩。它已经告诉读者这些词各自回答什么问题，但还不够像一段真正的前置教学，尤其当读者希望在进入 `02` 前多一点稳定感时，这一节的厚度仍然偏薄。

用户希望把这一节扩到接近“小型前置课”的力度，并且明确要求仍按仓库约定来写，而不是把 `00` 直接写成完整机器人动力学导论。

---

## 2. 设计目标

### 2.1 主目标

1. 保留 `00` 作为前三章入口的定位，不把它改写成独立大课。
2. 把 `articulation`、generalized coordinates、`M`、`J`、ABA、CRBA 这 6 个词从“一行表格”扩成更可读的正文块。
3. 让读者不仅知道“这个词是什么”，还知道“它和其他词怎么连起来”“在后面哪一章第一次真正展开”。
4. 保持文风通俗、短段、导航明确，和目前 `00/01/02` 的主线风格一致。

### 2.2 非目标

1. 不把 `00` 扩成正式的刚体动力学课程。
2. 不引入系统推导、成段公式或矩阵记号细节。
3. 不提前展开 `05_rigid_articulation` 与 `07_constraints_contacts_math` 的主体内容。
4. 不修改 `00` 的 Warp 小节和导航小节的定位，只做必要衔接。

---

## 3. 设计原则

### 3.1 从“词表”升级到“词与词之间的阅读链”

这次不只是把每个词写长一点，而是要让读者看到一条最小链路：

`articulation` -> generalized coordinates -> `M` / `J` -> ABA / CRBA

即使读者还不会推公式，也能知道这些名字不是散的，而是在同一套刚体阅读框架里依次出现。

### 3.2 每个词都回答四件事

每个词的展开正文都优先回答：

1. 它在解决什么问题？
2. 它和前后词是什么关系？
3. 你第一次在后续主线里会怎么碰到它？
4. 现在先懂到什么程度就够？

### 3.3 保留表格，但让表格退回“压缩总结”角色

仓库当前已经在 `00` 中使用表格压缩信息，这个结构可以保留，但不再让表格承担全部解释。正文先给阅读坡道，表格再负责压缩回顾。

### 3.4 不抢后续章节的戏份

`00` 这一节只做“第一次读懂这些词”的工作，不做：

- 刚体树算法细节
- 质量矩阵装配细节
- Jacobian 推导
- 约束数学细节

一旦正文开始逼近这些内容，就应主动收住，并指回 `05` 或 `07`。

## 5. 新的内容结构

`## 2. 刚体与机器人动力学最小词汇` 将被组织成下面这套结构：

1. **总引导段**
   - 说明这些词不是散点词表，而是一条刚体阅读链的最小骨架。
   - 明确你现在不需要会推导，只需要知道每个词在链条里负责什么。

2. **先讲整体，再讲单词**
   - 先用 1 小段解释：当系统从单个刚体变成“有关节的多刚体系统”时，为什么会自然需要坐标、惯性表示、速度映射、以及两类算法名字。

3. **逐词扩写**
   - `articulation`
   - generalized coordinates
   - mass matrix `M`
   - Jacobian `J`
   - ABA
   - CRBA

4. **关系收束段**
   - 用一小段把它们串起来：
     - 先有系统对象
     - 再有描述它的自由度坐标
     - 再有惯性和映射这类全局对象
     - 最后才轮到不同算法路径

5. **压缩总结表**
   - 保留表格作为复习工具，但表格只压缩“问题 / 角色 / 首次深入章节”。

## 6. 每个词的写法要求

### 6.1 `articulation`

- 重点写成“系统对象”概念
- 要强调它回答的是“我现在面对的是一个什么级别的机械系统”
- 要说明它是后面所有广义坐标、质量矩阵、Jacobian 讨论的前提容器

### 6.2 generalized coordinates

- 重点写成“描述系统姿态的最小坐标”
- 要和“直接盯世界坐标”形成对比
- 要让读者知道它为什么是 `M`、`J` 这类对象的共同语言

### 6.3 mass matrix `M`

- 重点写成“惯性如何在整套自由度里组织起来”
- 不写正式方程推导
- 可以用“哪些方向更难推动、哪些自由度会互相耦合”的直觉来讲

### 6.4 Jacobian `J`

- 重点写成“关节空间与任务/接触空间之间的映射桥梁”
- 说明它为什么经常和末端、接触点、约束联系在一起
- 不展开完整约束数学

### 6.5 ABA

- 重点写成“给定结构和输入时，高效做正向动力学”的算法名字
- 说明它是 Featherstone 路线里的关键词
- 让读者知道它是 `05` 的主角之一，而不是 `00` 里的推导对象

### 6.6 CRBA

- 重点写成“高效构造质量矩阵”的算法名字
- 说明它和 `M` 的关系要比和 solver 细节的关系更重要
- 让读者知道它常和 ABA 一起出现，但回答的问题不一样

## 7. 预期完成状态

扩写完成后，理想阅读效果应是：

1. 读者看见这 6 个词时，不再只会背单句定义。
2. 读者能用一两句话讲清它们为什么会一起出现在刚体主线里。
3. 读者进入 `02` 时，对 `Model / State / Control / Solver` 中会挂接哪些刚体词汇更有心理准备。
4. 读者进入 `05` 或 `07` 时，会觉得“这是前面提过的东西在真正展开”，而不是第一次见。

---

## 8. 风险与控制

### 风险 1：把 `00` 写成 `05` 的前传

控制：任何开始接近刚体树算法细节、空间代数细节、完整矩阵表达的段落都要收住，并转成“这一点到 `05` 再展开”。

### 风险 2：每个词都写了，但仍然像词典

控制：必须补一段“这些词怎么串起来”的总览，并在每个词里显式提到前后词的关系。

### 风险 3：正文变厚后，破坏 `00` 的快读节奏

控制：保持短段、强分节、结尾仍保留压缩总结表，让读者可以扫读也可以细读。
