# Chapters 00-02 Content Upgrade Design

- **日期**: 2026-04-18
- **仓库**: `/home/zhuzihou/dev/newton-learn`
- **范围**: `chapters/00_prerequisites`、`chapters/01_warp_basics`、`chapters/02_newton_arch`
- **目标**: 把前三章补成一条连续、可读、可进入 Newton 主线的最小教程起步包

---

## 1. 背景

当前仓库的结构、模板、规范和章节规划已经基本到位，但前三章正文还不平衡：

- `00_prerequisites` 已有种子内容，但更像速查页和跳转页，作为真正的前置入口还不够顺。
- `01_warp_basics` 还是模板状态，是当前 `M1` 的最大断点。
- `02_newton_arch` 已经有较完整的 seed content，但和 `00`、`01` 之间的衔接还不够紧，读者容易感觉“知道词了，但还没形成一路往下读的坡度”。

这轮工作不追求把前三章做成完整教材，而是优先把它们补成“能顺着读下去、能撑起后续章节”的主线入口。

---

## 2. 设计目标

### 2.1 目标

1. 让 `00 -> 01 -> 02` 形成连续阅读主线，而不是三个松散文件夹。
2. 让 `00` 承担更多 Newton 之外但后续立刻要用到的前置知识，且写得通俗、条理清楚。
3. 让 `01` 从模板升级为完整章节入口，并补出第一版 `principle.md`。
4. 让 `02` 成为第一个真正进入 Newton 本体的成型章节，而不是独立 seed note。
5. 保持克制，不把后续章节的主体内容提前塞进前三章。

### 2.2 非目标

1. 本轮不优先写 `source-walkthrough.md`。
2. 本轮不优先补 `examples.md`、`pitfalls.md`、`exercises.md`。
3. 本轮不以运行验证、性能记录或 `PROGRESS.md` 打勾为主目标。
4. 本轮不尝试系统讲完机器人动力学、GPU 编程、OpenUSD 或可微分仿真；只补进入 `01/02` 所需的最小前置。

---

## 3. 核心设计原则

### 3.1 先让人读懂，再追求覆盖度

前三章首先要解决“怎么顺利进入 Newton”的问题，不解决“把相关背景一次讲全”的问题。只要某个概念在 `01/02` 暂时够用，就先停在够用的位置。

### 3.2 把 `00` 写成真正的前置课，而不是百科索引

`00` 要从“只负责查词和跳转”调整为“帮助读者建立最低限度的阅读直觉”。写法上优先回答：

- 这个概念是干嘛的？
- 你读后两章时会在哪遇到它？
- 现在只需要懂到什么程度？

### 3.3 `01` 不重复 `00`，只承接并推进

`00` 给出 GPU/Warp 最小直觉，`01` 才进入 Warp 的对象、执行模型和 Newton 中的使用方式。避免把同一段解释在两章复制两遍。

### 3.4 `02` 负责第一次真正落到 Newton 本体

`02` 不是再讲一遍前置知识，而是把 `00/01` 的基础落到 `basic_pendulum`、`newton.examples`、`Model / State / Control / Solver` 和 solver family 的整体地图上。

### 3.5 写作风格偏通俗、短段、强导航

每节优先采用：

1. 为什么需要这节
2. 最小直觉解释
3. 在后续哪一章第一次真正展开

避免一上来堆公式、术语表或长列表定义。

## 4. 文件级改动设计

### 4.1 `chapters/00_prerequisites/README.md`

把当前 README 从“轻量入口说明”改成“前三章总入口页”，但仍保留它是前置章节而不是独立大课的定位。

计划内容：

- 更明确地解释本章的作用：不是学完所有前置，而是保证读者能进入 `01/02`
- 重写完成门槛，使其更像可操作的阅读结果，而不是抽象自查
- 重写“本章目标”，明确三类前置：动力学词汇、GPU/Warp 直觉、仿真一步的通用框架
- 优化阅读顺序，让读者知道为什么先读 `00`，读完后怎样进入 `01` 再进入 `02`
- 保留“GAMES103 已有 vs 本章新增”，但把它写得更具体、更面向后三章实际用途

### 4.2 `chapters/00_prerequisites/principle.md`

这是本轮对 `00` 的重点改写文件。它会从“速查表主导”扩成一份更通俗的前置小课。

建议结构：

1. **为什么先有这一章**
   - 解释学习者当前的真实处境：已经懂一些物理仿真，但不熟 Newton、Warp、机器人动力学词汇
   - 说明本章只补最小必需前置

2. **先建立仿真一步的通用直觉**
   - 用不依赖 Newton 的语言讲清“世界描述、当前状态、外部输入、做一步更新”
   - 给 `02` 的四层结构打地基，但不提前用过多 Newton 术语

3. **刚体与机器人动力学最小词汇**
   - `articulation`
   - generalized coordinates
   - mass matrix `M`
   - Jacobian `J`
   - ABA / CRBA
   - 每个词都用“它解决什么问题”来写，不展开推导

4. **GPU / Warp 最小直觉**
   - 从 CPU for-loop 到“很多元素同时做同一件事”的思路切换
   - 为什么需要 `kernel`
   - 为什么需要设备数组
   - 为什么会出现 `atomic_add`
   - `tile` 和 `Graph` 只给足够第一遍阅读用的直觉，不提前深入优化细节

5. **当前缺口如何落到 01 和 02**
   - 从“读者现在可能卡住的问题”出发，告诉他先去 `01` 还是先去 `02`
   - 让跳转关系更自然，而不是只给一个表格

写法要求：

- 减少单纯词表堆叠，增加短解释段
- 保留表格，但表格只作为压缩总结，不作为唯一解释方式
- 避免写成“百科导读”或“机器人动力学绪论”

### 4.3 `chapters/01_warp_basics/README.md`

把当前模板内容替换为完整章节入口。

计划内容：

- 明确本章真正回答的问题：为什么 Newton 学习者需要先懂 Warp 的最小编程模型
- 明确前置依赖只要求 `00` 的最小内容，不要求完整 CUDA 背景
- 明确最小闭环：读者能把 `for-loop`、`kernel`、`array`、`launch`、`tid`、`atomic` 放进一个稳定心智模型
- 写清本章完成后怎样帮助 `02` 和后续 `11`、`13`

### 4.4 `chapters/01_warp_basics/principle.md`

新增该文件，作为 `01` 的主体内容。

建议结构：

1. **为什么 Newton 学习里必须先补 Warp**
2. **从 CPU for-loop 到 kernel**
3. **`wp.array` 在心智模型里是什么**
4. **`wp.launch`、`wp.tid()` 和批量执行到底是什么关系**
5. **为什么会遇到 `wp.atomic_add`**
6. **怎么理解 `wp.tile` 与局部复用**
7. **怎么把 `Graph` 理解成“重复执行的调度打包”**
8. **带着这些概念进入 Newton 时，应该重点看什么，不必急着看什么**

写法要求：

- 避免直接照抄 Warp API 文档
- 每节都要回扣到“Newton 学习者为什么要懂它”
- 不承诺这章就能覆盖所有 Warp；保持对后续 `11_mpm` 和 `13_diffsim` 的接口感

## 5. 章节间叙事链

目标叙事链如下：

1. `00` 回答：为了读懂 Newton，先要补哪些最小前置？
2. `01` 回答：如果 Newton 建在 Warp 上，那 Warp 的最小执行模型到底是什么？
3. `02` 回答：带着这些前置，Newton 自己的 examples、对象分层和 solver 地图是怎样组织起来的？

如果这条链成立，那么读者读完前三章后应该达到的状态是：

- 见到 `articulation`、`M`、`J`、kernel、array、atomic 这些词不会发懵
- 能接受 Newton 是“基于 Warp 的多 solver 家族”，而不是单个 demo 仓库
- 能用 `basic_pendulum` 作为第一个入口理解 Newton 的总体组织方式

## 6. 实施顺序

为了保证前三章真正连起来，实施顺序按以下方式进行：

1. 先重写 `00_prerequisites/principle.md`
2. 再调整 `00_prerequisites/README.md`
3. 再补 `01_warp_basics/README.md`
4. 新增并写完 `01_warp_basics/principle.md`
5. 最后回头调整 `02_newton_arch/README.md` 与 `principle.md`

理由：

- `00` 定位不稳，`01` 很容易写成又一份前置课
- `01` 不立起来，`02` 和前两章之间的连接就仍会悬空
- 先把前置和 Warp 讲清，才能知道 `02` 里哪些地方该加承接、哪些地方该删重复

## 7. 成功标准

本轮完成后，应满足以下标准：

1. `00` 读起来更像“真正的前置小课”，而不是词表和跳转页的组合。
2. `01` 不再是模板，且有可读的 `README.md` 与 `principle.md`。
3. `02` 与 `00/01` 的衔接明显增强，读者能感到三章之间存在连续坡度。
4. 三章都保持克制，没有把后续章节主体内容提前写爆。
5. 文风统一：通俗、短段、少堆术语、始终说明“这和后面哪一章有关”。

## 8. 风险与控制

### 风险 1：`00` 膨胀成独立大课

控制：凡是不能直接帮助进入 `01/02` 的背景内容，都不展开；必要时只留一句点到为止。

### 风险 2：`01` 变成 Warp 官方文档摘要

控制：每节必须回答“Newton 学习者为什么现在要懂这个”，而不是纯 API 罗列。

### 风险 3：`02` 重复 `00/01`

控制：`02` 只承接前置，不回头重讲大段 GPU/动力学背景；重点放在 Newton 本体对象和 examples 入口。

### 风险 4：叙述过于抽象，仍然不够好读

控制：优先短段、对比、直觉化表达；表格只做总结，不承担全部解释任务。
