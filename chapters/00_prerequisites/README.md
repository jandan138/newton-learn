# 00 前置速查与深入指针

这章不是一门把机器人动力学、GPU 编程和 Warp 全部讲完的独立课程。它的任务更窄一些：先把你安全送进 `01_warp_basics` 和 `02_newton_arch`，让前三章形成一条连续可走的入口。

如果你已经认得这些前置词汇，这章就应该读得很快。把它当成导航页和补洞页，而不是新的学习终点。

## 完成门槛

```text
[ ] 我已经判断出自己现在更该先读 `01_warp_basics` 还是 `02_newton_arch`
[ ] 我已经记下 3-5 个当前不稳、但后两章马上会遇到的词，并知道它们第一次在哪章真正展开
[ ] 如果这些词我已经熟悉，我已经把本章当成快速导航读完，并继续进入 `01` 或 `02`
```

## 本章目标

- 先用最小篇幅补三类马上会用到的前置：仿真一步的通用框架、机器人动力学常见词、Warp 基础词汇。
- 帮你把“我到底卡在哪”落成具体问题，再把它接到 `01` 和 `02` 的第一个深入位置。
- 把这章作为前三章的入口使用，而不是停在这里把所有背景一次学完。

## 前置依赖

- 无。
- 如果你已经有 GAMES103 或普通物理仿真的基础，这章可以快速通过。
- 不要求你先通读完整的动力学教材、CUDA 教程或 Warp 文档；缺什么，再补什么。

## GAMES103 已有 vs 本章新增

| 维度 | GAMES103 已有 | 本章新增 |
|------|----------------|----------|
| 仿真通用直觉 | 已经知道仿真会更新状态、处理约束和输入。 | 把这种直觉压缩成能直接带进 `02` 的“描述世界 / 保存状态 / 施加输入 / 推进一步”四格框架。 |
| 机器人动力学词汇 | 对刚体、关节、约束、质量矩阵有课程级直觉。 | 补 `articulation`、`M`、`J`、ABA、CRBA 这些后面会直接出现的词，并指出它们第一次在哪章真正展开。 |
| GPU / Warp 视角 | 对 GPU 编程模型通常不是主线。 | 先给 `kernel`、`array`、`launch`、`atomic_add`、`tile`、`Graph` 的最小直觉，方便进入 `01`。 |

## 阅读顺序

1. 先快速判断你现在更缺哪一块：仿真一步框架、动力学词汇，还是 Warp 执行直觉。
2. 只补当前最影响阅读的一两块，不要把本章当成必须整章精读的大课。
3. 进入 `01_warp_basics`，把 Warp 词汇从“听过”推进到“有稳定心智模型”；如果这些词你已经熟，就快速通过 `01`。
4. 再进入 `02_newton_arch`，把前两章的词和直觉落到 Newton 的 `Model / State / Control / Solver` 上。
5. 后面读例子或源码时，只要再次卡词，再回本章对应小节补洞即可。
