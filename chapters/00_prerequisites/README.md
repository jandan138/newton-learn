# 00 前置速查与深入指针

## 完成门槛

```text
[ ] 我已经识别出当前最大的前置缺口，而不是泛泛地说“基础都不太稳”
[ ] 我已经根据这个缺口跳到第一个对应的深入章节，而不是试图把本章整章刷完
[ ] 我至少用本章解决过一次真实困惑，并把它记成后续回看入口
```

## 本章目标

- 把 `00_prerequisites` 作为轻量入口，用来快速补词汇、补章节导航，而不是把机器人动力学、Warp 或接触数学一次讲完。
- 帮你先判断“我现在卡在什么缺口”，再把这个缺口映射到后续第一次真正展开的章节。
- 在开始 `02` 及后续章节前，先守住最小可用前置，而不是把大背景扩成独立课程。

## 前置依赖

- 无。
- 后续遇到具体缺口时，再按需阅读对应章节里的 `[MUST]` references；本章本身不要求先通读任何大部头。

## GAMES103 已有 vs 本章新增

| 维度 | GAMES103 已有 | 本章新增 |
|------|----------------|----------|
| 机器人动力学词汇 | 对刚体、关节、约束、质量矩阵有课程级直觉。 | 补 `articulation`、`ABA`、`CRBA`、`Delassus operator` 这类后续章节会直接出现的英文词表和首次深入位置。 |
| Newton 学习路径 | 课程内容按主题展开，不负责 Newton 内部阅读顺序。 | 明确“某个缺口第一次在哪一章真正展开”，避免前置阅读失控。 |
| GPU / Warp 视角 | 对 GPU 编程模型和可微分物理通常不是主线。 | 给出 `wp.kernel`、`wp.array`、`wp.launch`、`Tape/adjoint` 等一页速查，足够支撑进入 `01`、`11`、`13`。 |

## 阅读顺序

1. 只读与你当前缺口对应的小节，不要试图一次读完整章。
2. 读完后立刻跳去对应的第一个深入章节，把速查表里的词放回真实上下文。
3. 当这个词或概念第一次在例子、源码或 paper 里真的出现时，再回到本章复看一次，确认自己补对了洞。

## 预期产出

- 一份个人前置缺口清单，至少写下当前最不稳的 3 个词或主题。
- 一张缺口到后续章节的映射：每个缺口都要能落到一个“第一次深入”的章节，而不是停留在抽象焦虑。
